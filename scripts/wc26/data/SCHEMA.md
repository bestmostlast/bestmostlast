# WC26 data model — team history, players, H2H

Three CSV types. **Team data is per-team and reused** across every match that team
plays; **H2H is derived per game** from two teams' data. So we source ~48 teams once,
not 104 matches.

```
data/
├── teams/
│   ├── MEX.matches.csv     match-by-match WC history (one row per WC match played)
│   ├── MEX.players.csv     player WC stats (one row per player)
│   ├── RSA.matches.csv
│   ├── RSA.players.csv
│   └── … (one pair per nation, FIFA 3-letter code)
└── h2h/
    └── m001-mexico-vs-south-africa.csv   derived: the two teams' past WC meetings
```

## Source (SOLVED)
All team files are generated from the **Fjelstul World Cup Database**
(github.com/jfjelstul/worldcup, MIT license) — every men's WC match 1930→2022.
Vendored at `data/source/fjelstul_matches.csv` (+ `_goals.csv`). Transform:
`build_team_data.py`. One file → all 84 nations, match-by-match. Verified:
Mexico P60 17-15-28 GF62 GA101, Brazil P113, Ronaldo 15 / Pelé 12 WC goals.
Re-run `python3 scripts/wc26/build_team_data.py` to rebuild.

(Historical note: an earlier hand-built pass used per-edition `EDITION-SUMMARY` rows
when match data wasn't sourced. That's now obsolete — every team is full match-level.)

## teams/<CODE>.matches.csv
One row per World Cup match the team played (or per-edition summary).

| column | meaning | example |
|---|---|---|
| code | FIFA 3-letter | RSA |
| team | display name | South Africa |
| year | WC edition | 2010 |
| stage | group / r16 / qf / sf / final / third / EDITION-SUMMARY | group |
| match_no_wc | sequential WC match # for this team (1..N) | 7 |
| opponent | opponent name | Mexico |
| opponent_code | FIFA code | MEX |
| gf | goals for | 1 |
| ga | goals against | 1 |
| result | W / D / L | D |
| host | 1 if team hosted that edition else 0 | 1 |
| note | optional | opening match |

Cumulative totals (for the race chart) are DERIVED from this by the generator —
never stored here. This is the long source of truth.

## teams/<CODE>.players.csv
One row per player's WC career (aggregate across tournaments).

| column | meaning | example |
|---|---|---|
| code | FIFA code | RSA |
| player | name | Benni McCarthy |
| wc_goals | total WC goals | 2 |
| wc_assists | total WC assists (modern only; blank if unknown) | |
| wc_apps | WC appearances | |
| years | tournaments played | 1998;2002 |
| note | optional | |

## h2h/<slug>.csv  (DERIVED per game)
The past WC meetings between the two teams in this fixture. Generated by matching
each team's `.matches.csv` opponent rows. One row per past meeting + a summary header
is implied by aggregating.

| column | meaning | example |
|---|---|---|
| year | edition they met | 2010 |
| stage | stage | group |
| team_a | fixture's team A | Mexico |
| team_b | fixture's team B | South Africa |
| score_a | A's goals | 1 |
| score_b | B's goals | 1 |
| result_for_a | W / D / L from A's view | D |
| note | | opening match, Tshabalala goal |
```
